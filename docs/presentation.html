spyc 🌶️

A Rust TUI file commander where the AI agent
in the side pane can query the file commander itself.

The file commander is the noun the agent operates on, not the chrome around it.

Derek Marshall May 2026

What is spyc?

A two-pane terminal program inspired by SideFX's spy: vim-flavoured file commander on top, a child process below (Claude Code, codex, gemini — all first-class). They share focus and the file commander exposes a local MCP socket the agent connects to.

🔴 The Problem

  • "AI in your terminal" tools give the agent a chat window and copy-pasted paths
  • Agent has no view of what the user is looking at
  • "Which file?" gets asked every other turn
  • Context switching between file browser and agent kills flow

🟢 The Solution

  • File list + agent pane in one terminal session
  • MCP socket lets the agent ask: what is the cursor on, what is staged, what is picked
  • Bidirectional: gf jumps from agent output to the file list
  • Session restore resumes each pane to its own agent conversation

🔗 The MCP Bridge

This is what makes spyc more than "a terminal inside a terminal."

How It Works

  • spyc starts an MCP server on a PID-scoped Unix domain socket
  • Agents auto-discover via .mcp.json (Claude) / .codex/config.toml (codex) / settings.json (gemini) — no flags needed
  • Both registrations re-exec spyc --mcp as a stdio proxy that forwards to the same socket
  • Multiple instances coexist; takeover is explicitly prompted on stderr (PID N already owns MCP here. Take over?)
  • Context file updated every event loop tick
  • Hand-rolled JSON-RPC 2.0 — no async runtime

What Claude Sees (read)

  • 📂 Current directory
  • 📄 Cursor file + file contents
  • 📌 Picks (selected files)
  • 📦 Inventory (yanked file cache)
  • 🔍 Active filter
  • 🌿 Git branch + dirty state

What Claude Can Do (write)

Claude can mutate the TUI via MCP tools — command channel from server thread to event loop:

navigate_to set_filter pick_files clear_picks get_file_content

"Filter to Rust files and pick the tests" → Claude calls set_filter + pick_files → TUI updates live.

🏗️ Architecture

Clean module boundaries. One event loop. Action-driven dispatch.

src/app/ Event loop, layout, key dispatch, state management
src/keymap/ Action enum (full verb vocabulary), resolver, user keymap DSL
src/pane/ Pty subprocess, multi-tab, vt100 rendering, ANSI key encoding
src/mcp.rs Unix-socket MCP server + stdio proxy for agent auto-discovery, JSON-RPC 2.0 dispatch
src/context.rs Context snapshot (cwd, cursor, picks, git) written every tick
src/ui/ List view, pager, status bar, prompt, line editor, theme
src/state/ Cursor, marks, picks, inventory (file cache), sessions, history
src/fs/ Directory listing, file operations
src/config/ TOML config with keymap DSL, live reload
src/shell/ Shell expansion, command execution

Every user action maps to an Action enum variant. New features = new variant + keymap binding + handler.

⚙️ Design Patterns

Action Dispatch

Single Action enum is the full vocabulary of user behaviour. Keymap DSL binds any key to any action. One match arm per feature.

Dirty-frame Rendering

Event-driven: 0 dps at idle. DEC 2026 synchronized output wraps every frame. build_rows() + grid cached via generation counter. ~2.5% idle CPU (vim-competitive).

Pty Multiplexing

Each tab is an independent pty. vt100 parser processes output. Bracketed paste forwarding. 10K-line scrollback per tab.

MCP Context + Commands

Read: atomic JSON snapshot, throttled 1/sec. Write: command channel (mpsc) from MCP socket threads to event loop with one-shot reply channels. 5s timeout.

Vi Everywhere

Line editor has insert/normal modes, operator+motion (dw, cw), word objects. Same muscle memory from prompt to pager to file list.

File-backed Inventory

Yanked files copied to ~/.local/state/spyc/inventory/ with UUID metadata. Graveyard for safe deletes. Persists across sessions.

  • Copy, move, remove, chmod, mkdir
  • Search with glob patterns
  • Named marks with m{a-z} / '{a-z}

  • MCP bridge — Claude reads and writes workspace state
  • gf / gF — jump from pane output to file list
  • Context piping: ^a P sends file contents to pane
  • Session save/restore with Claude conversation resume
  • 10K-line scrollback with save
  • Tab rename, activity indicators

📖 In-App Pager

  • Syntax highlighting (syntect), ANSI colors, streaming
  • Search, line numbers, hex dump, line wrapping
  • Buffer history: [b / ]b navigate closed pagers
  • v opens in $EDITOR, returns to pager

🌿 Git Integration

  • Branch + dirty flag in powerline status bar
  • File status colors (modified, untracked, added, deleted)
  • g d / g D diff view in pager
  • Worktree list, create, switch, delete

🤖 Building with Claude Code

"spyc is both a product of Claude Code and a host for it. The developer uses spyc with Claude running in the lower pane — bugs and features are discovered through this dog-fooding loop."

What Worked

  • Milestone-driven development: clear scope per feature
  • Action enum pattern: Claude adds features without breaking existing ones
  • Docs-in-sync convention: ROADMAP, FEATURES, CHANGELOG updated every commit
  • Dog-fooding: running Claude inside spyc surfaces UX issues fast

What's Interesting

  • 35K+ lines of idiomatic Rust (clippy pedantic + nursery, zero warnings)
