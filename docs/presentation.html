What is cspy?

A terminal file manager inspired by SideFX's spy, rebuilt for modern developer workflows — specifically designed to pair with AI coding assistants.

The Problem

  • Terminal workflows are fragmented: ls, cd, vim, git, tmux
  • AI coding assistants run in a separate window
  • Context switching kills flow
  • Reaching for the mouse, Finder, or Cmd+Tab breaks your rhythm

The Solution

  • Hands on home row — never leave the keyboard
  • Browse files in the top pane, talk to Claude below
  • Pipe file contents directly into the AI
  • Vi keybindings everywhere — muscle memory transfers

Nothing like this exists

We searched. Here's what's out there:

Yazi / Joshuto / Felix

Rust TUI file managers with vim keybindings. Fast, popular (Yazi has 33k+ stars).

No AI integration. No embedded pty.

Xplorer

Rust + Tauri GUI file manager. Has Claude Code terminal, split panes, code editor.

GUI app, not terminal-native. Electron-class weight.

Wave / TmuxAI

Wave: terminal with built-in AI. TmuxAI: AI overlay that reads tmux panes.

Terminals with AI bolted on. Not file managers.

cspy occupies an empty niche: terminal-native file manager
with embedded AI pty and context piping.

Target Platforms

Built with ratatui + crossterm + vt100. Static Linux binaries via musl. macOS universal binary.

Architecture

Clean module boundaries. One event loop. Action-driven dispatch.

src/app.rs Event loop, layout, all key dispatch (3,000 lines)
src/keymap/ Action enum, resolver, user keymap DSL
src/pane/ Pty-hosted subprocess, multi-tab, vt100 rendering
src/ui/ List view, pager, status bar, prompt, line editor, theme
src/fs/ Directory listing, file operations
src/state/ Cursor, marks, picks, inventory, history
src/config/ TOML config with keymap DSL, live reload
src/shell/ Shell expansion, command execution

Every user action maps to an Action enum variant. New features = new variant + keymap binding + handler.

Design Patterns

Action Dispatch

Single Action enum is the full vocabulary of user behaviour. Keymap DSL binds any key to any action. One match arm per feature.

One-shot Repaint

needs_full_repaint flag instead of per-frame terminal.clear(). Set on overlay/pager close. Eliminates diff-buffer ghost artifacts.

Pty Multiplexing

Each tab is an independent pty. vt100 parser processes output. Cursor blink via software toggle. Bracketed paste forwarding.

Vi Everywhere

Line editor has insert/normal modes, operator+motion (dw, cw), word objects. Same muscle memory from prompt to pager to file list.

Live Config

.cspyrc.toml with fs watcher. Per-user and per-project. Keymap DSL, color overrides, ignore mask patterns. No restart needed.

Context Piping

^W p reads file contents and sends to the pane as bracketed paste with [file: path] headers. Bridge between browsing and prompting.

Feature Highlights

File Management

  • Multi-column listing with vi navigation
  • Picks (per-dir) + inventory (cross-dir)
  • Copy, move, remove, chmod, mkdir
  • Search with glob patterns
  • Named marks with m{a-z} / '{a-z}
  • Ignore masks (dotfiles, build artifacts)

Split Pane

  • Multi-tab pty (Claude, zsh, anything)
  • Tab rename, activity indicators
  • 10K-line scrollback with save
  • Context piping: file contents to pane
  • Focus indicators: cursor blink + dim
  • Alt+Enter for multi-line input
  • Dedicated pane command history with dedup
  • Session save/restore (--resume)

In-App Pager

  • ANSI color preservation, streaming output
  • Search, line numbers, hex dump
  • Multi-column layout, position indicator
  • v opens in $EDITOR, returns to pager

Git Integration

  • Branch + dirty flag in status bar
  • File status colors (modified, untracked, etc.)
  • g d / g D diff view in pager
  • Worktree list, create, delete
  • Powerline-style status segments

Building with Claude Code

"cspy is both a product of Claude Code and a host for it. The developer uses cspy with Claude running in the lower pane — bugs and features are discovered through this dog-fooding loop."

What Worked

  • Milestone-driven development: clear scope per feature
  • Action enum pattern: Claude adds features without breaking existing ones
  • Docs-in-sync convention: ROADMAP, FEATURES, help text updated with every commit
  • Iterative refinement: ship, dog-food, fix, repeat

What's Interesting

  • 12,800 lines of idiomatic Rust (clippy pedantic + nursery)
  • Complex terminal state: pty multiplexing, vt100 parsing, cursor management
  • Subtle UX: focus indicators, ghost char fixes, blink timing
  • Cross-platform: macOS universal + Linux static binaries
